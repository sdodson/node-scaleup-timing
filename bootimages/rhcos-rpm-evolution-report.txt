KEY FINDINGS
================================================================

1. CONSISTENT UPDATE RATE: Across all 5 major versions, roughly
   62-67% of packages remain at their GA version by the final
   z-stream, while 31-37% get updated. This ratio is remarkably
   stable regardless of how many z-streams a release has (29 to 87).

1a. CROSS-MAJOR SIMILARITY: Comparing 4.16.0 to 4.18.38 (which
   share the RHEL 9.4 base) yields almost the same ratio as the
   within-version 4.16.0→4.16.60 comparison: 61% unchanged vs 62%.
   The 205 updated packages are the same count in both cases.
   The main difference is 23 newly added packages in 4.18 (mostly
   python3 modules, subscription-manager, and OCP runtime components).
   This suggests the RHEL base version is the dominant factor in
   determining which packages change, not the OCP major version.

2. PACKAGE COUNT GROWTH: The RHCOS image has grown steadily from
   503 packages (4.12) to 570 packages (4.19), adding ~67 packages
   over 4 major versions.

3. MINIMAL CHURN: Package additions and removals within a z-stream
   lifecycle are very rare (0-4 per cycle). The package manifest
   is essentially fixed at GA.

4. OPENVSWITCH ROTATION: Every release cycle swaps openvswitch
   versions (2.17→3.1→3.3→3.4→3.5), always done mid-z-stream.
